Privacy & Security Policy

FlashDrop Pro is built with a single principle: your data never belongs to us. This document explains exactly how FlashDrop Pro handles privacy, security, and data protection — in plain, verifiable terms.

1. Pure Peer-to-Peer Architecture

FlashDrop Pro uses WebRTC-based peer-to-peer (P2P) connections. Files and text are transferred directly between browsers.

  • No file data is uploaded to or stored on our servers
  • No cloud storage, no intermediaries
  • Once the browser tab is closed, all session data is destroyed

2. Zero Server Storage

FlashDrop Pro does not store:

  • Files or file fragments
  • Text content
  • Download history
  • Transfer metadata

Our servers are used only for minimal connection signaling — never for content delivery.

3. Zero Logs Policy

We do not log:

  • IP addresses
  • User identities
  • File names or sizes
  • Transfer timestamps

4. End-to-End Session Control

Users can define strict access limits:

  • One-time download
  • Limited number of downloads
  • Unlimited (until sender closes the session)

Once the limit is reached, the session self-destructs automatically.

5. Encryption & Transport Security

All transfers occur over encrypted WebRTC channels using modern browser security standards. Data is encrypted in transit and never decrypted by FlashDrop Pro servers.

6. What We Can — and Cannot — See

  • We can see: anonymous connection requests
  • We cannot see: file content, text, names, or user identity

7. Cookies & Tracking

FlashDrop Pro does not use tracking cookies, analytics identifiers, or fingerprinting scripts.

8. Legal & Compliance

Because FlashDrop Pro does not process or store user data, most data protection regulations (GDPR, CCPA) do not apply in the traditional sense.

9. User Responsibility

Users are responsible for the content they choose to share. FlashDrop Pro cannot access, moderate, or recover transferred data.

10. Transparency

FlashDrop Pro is designed to be auditable by behavior: if a server never receives your files, it cannot leak them.

隐私与安全说明

FlashDrop Pro 的核心原则只有一个：你的数据永远不属于我们。

本页面将以清晰、可验证的方式说明 FlashDrop Pro 如何处理隐私、安全与数据保护。

1. 纯 P2P 架构

FlashDrop Pro 基于 WebRTC 实现浏览器之间的点对点（P2P）连接。 文件和文本直接在设备之间传输。

2. 零服务器存储

  • 不存储文件
  • 不存储文本
  • 不存储下载记录
  • 不存储传输元数据

3. 零日志政策

  • 不记录 IP
  • 不记录用户身份
  • 不记录文件名或大小

4. 会话自毁机制

每次传输都是临时会话，达到下载限制或关闭页面后即自动销毁。

5. 加密传输

所有数据通过 WebRTC 加密通道传输，服务器无法解密或查看内容。

6. 我们能看到什么

我们只能看到匿名连接请求，无法看到任何内容数据。

プライバシーとセキュリティ

FlashDrop Pro は「データはユーザーのもの」という原則で設計されています。

すべての転送はブラウザ間の P2P 接続で行われ、サーバーに保存されることはありません。
